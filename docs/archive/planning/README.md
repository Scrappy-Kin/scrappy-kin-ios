# Planning Archive

`docs/archive/planning/` is archive-only.

Rules:
- Do not create new `active/`, `backlog/`, or `NOW.md` planning surfaces here.
- Treat files here as historical context, not the active source of truth.
- Put live execution state in `THREAD_A_STATE.md`, `workstreams/`, and the relevant local handoff pack.
- Put active UI reasoning in `docs/ui/design-decisions-log.md`.
- If a historical plan still matters, reference it from a current workstream instead of reviving the old planning system.

Current structure:
- `archive/legacy-active/`: old active planning docs kept for traceability
- `archive/legacy-backlog/`: old backlog docs kept for traceability
- `archive/deprecated/`: retired planning scaffolding and stale control files

Cleanup note:
- If a future pass needs one of these docs, either migrate the useful logic into a current workstream/doc or leave it archived. Do not reopen the old folder model.
